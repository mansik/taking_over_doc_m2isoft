		CIS 서버의 파일 백업 :CIS 서버의 파일 - > ynology cloud station backup 프로그램 - > NAS 로 백업

			(1) '#장비 이미지 경로'로 이미지, 결과파일 생성
				<'#장비 이미지 경로'> 폴더 구조
				[pacs_image] : 장비에서 저장되는 이미지 및 결과파일, 폴더명이 [VSDATA] 등 장비마다 다를 수 있음(CONF.INI에 "EQPM_IMAGE_PATH = "의 폴더)
					 : SERIAL 통신인 경우 M2I_CISIF 프로그램에서 이미지 및 결과파일 생성함.	
					 : C:\, 또는 d:\에 존재
		2-2)장비인터페이스 프로그램이 없는 경우 -> 시리얼 통신
			(1) M2I_SOFTIF 프로그램에서 시리얼 통신을 통해 
			(2) '#장비 이미지 경로'로 이미지, 결과파일 생성
		3)M2I_SOFTIF 프로그램
			(0)conf.ini 를 통한 환경설정 파일 읽음
			(1)TIMER를 사용하여 '#장비 이미지 경로'에서 '#로컬 이미지 경로'로 이미지 및 결과 파일을 [M2I_SOFT] - [장비코드] 폴더로 파일명을 아래와 같이 변경해서 복사			
				<'#로컬 이미지 경로' 폴더 구조>
				[M2I_SOFT] : 장비에서 받은 결과 파일이 저장되는 폴더(이미지 파일(jpg), 결과파일(txt, csv) :  확장자는 conf.ini에 "LOCAL_IMAGE_PATH = "에 설정
					- [BCA] : 장비코드 (conf.ini에서 설정)별 폴더가 프로그램 실행시 자동을 생성됨
						: [pacs_image]폴더의 파일을 TimerSendMakeImageFileName 에 의해서 3초 간격으로 이 폴더로 이동됨
						: 이동된 파일명 :  
							:등록번호13자리_20191203_PW_0.jpg : 이미지파일명, 이미지 1장인 경우
							:등록번호13자리_20191203_PW_1.jpg : 이미지파일명, 이미지 여러장인경우
							:등록번호13자리_20191203_PW_0.csv : 결과파일명 (장비에서 csv로 생성됨)
							:등록번호13자리_20191203_PW_0.txt: 결과파일명 (장비에서 txt로 생성 또는 동맥경화의 경우 이미지 파일에서 결과 추출할 경우 Zan Printer 에서 txt로 결과 변환해서 생성)
						: conf.ini의 IMAGEFILENAME_MAKE_YN = Y인 경우 M2I_CISIF 프로그램에서 [전송]버튼을 눌러야 서버에 저장됨
						: conf.ini의 IMAGEFILENAME_MAKE_YN = N인 경우 자동으로 서버에 저장됨
					  - [Send]	: 서버로 전송 + 병원DB에 저장 성공한 파일이 이동, 자동으로 폴더 생성
					  - [NotSend] : 서버로 전송 + 병원DB에 저장 실패한 파일이 이동, 자동으로 폴더 생성, 
								: 해당 이미지와 결과파일을 [장비코드]폴더의 루트로 이동하면 다시 서버로 전송을 실행할 수 있다.			
			(2) 파일을 CIS서버로 FTP로 전송 : conf.ini파일에 따라 자동, 수동 전송 여부 결정 
			(2-1) IMAGEFILENAME_MAKE_YN = Y 인 경우 : 화면에 이미지, 결과 파일을 보이지 않고 자동 전송
			(2-2) IMAGEFILENAME_MAKE_YN = N 인 경우 : 화면에 이미지, 결과 파일을 보여주고 사용자가 환자를 선택하여 전송

			(3-1) 정상적으로 전송된 경우
				:1. 병원DB update
				:2.  [M2I_SOFT] - [장비코드] 내 파일을 [M2I_SOFT] - [장비코드] - [Send] 로 이동
			(3-2) 전송이 안된 경우
				:1.  [M2I_SOFT] - [장비코드] 내 파일을 [M2I_SOFT] - [장비코드] - [NotSend] 로 이동			 
			
		4) FTP 서버
			(1) 전송된 파일을 synology NAS 로 실시간 백업(synology cloud station backup  프로그램 사용)
